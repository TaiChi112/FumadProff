# ch11: นิยาม P, NP, Co-NP, NP-Complete, NP-Hard และความสัมพันธ์

## 1) นิยามแต่ละกลุ่มปัญหา

### 1.1 P
กลุ่มปัญหาแบบตัดสินใจ (decision problems) ที่แก้ได้ในเวลาเชิงพหุนามด้วยเครื่องแบบกำหนดแน่นอน (deterministic machine)

ตัวอย่าง: Shortest Path (ฉบับ decision), Minimum Spanning Tree (ฉบับ decision)

### 1.2 NP
กลุ่มปัญหาแบบตัดสินใจที่ถ้าคำตอบเป็น Yes จะมีพยาน (certificate) ที่ตรวจสอบได้ในเวลาเชิงพหุนาม

ตัวอย่าง: SAT, Hamiltonian Cycle (decision), Clique (decision)

### 1.3 Co-NP
กลุ่มปัญหาที่ส่วนเติมเต็ม (complement) อยู่ใน NP
หรือกล่าวอีกแบบคือ ถ้าคำตอบเป็น No จะมีพยานที่ตรวจสอบได้ในเวลาเชิงพหุนาม

ตัวอย่าง: UNSAT เป็นปัญหาส่วนเติมเต็มของ SAT

### 1.4 NP-Complete
ปัญหาใน NP ที่ยากที่สุดใน NP
มีเงื่อนไขพร้อมกัน 2 ข้อ
1. ปัญหาอยู่ใน NP
2. ทุกปัญหาใน NP สามารถลดรูปแบบพหุนามมาที่ปัญหานี้ได้

ตัวอย่าง: SAT, 3-SAT, Vertex Cover (decision), Clique (decision)

### 1.5 NP-Hard
ปัญหาที่ยากอย่างน้อยเท่ากับปัญหาใน NP-Complete
ไม่จำเป็นต้องเป็นปัญหา decision และไม่จำเป็นต้องอยู่ใน NP

ตัวอย่าง: TSP แบบ optimization, Halting Problem (ยากมากและไม่อยู่ใน NP)

## 2) ความสัมพันธ์ระหว่างกลุ่ม

1. ความสัมพันธ์พื้นฐาน

$$
P \subseteq NP, \qquad NP\text{-Complete} \subseteq NP, \qquad NP\text{-Complete} \subseteq NP\text{-Hard}
$$

2. Co-NP คือ complement-side ของ NP

$$
L \in Co\text{-}NP \iff \overline{L} \in NP
$$

3. สถานะที่ยังไม่ทราบแน่ชัดในปัจจุบัน
- ยังไม่รู้ว่า P เท่ากับ NP หรือไม่
- ยังไม่รู้ว่า NP เท่ากับ Co-NP หรือไม่

## 3) ภาพรวมเชิงอธิบาย
1. P คือปัญหาที่ทั้งแก้ได้เร็วและตรวจได้เร็ว
2. NP คือปัญหาที่ตรวจคำตอบได้เร็ว
3. NP-Complete คือตัวแทนความยากสูงสุดใน NP
4. NP-Hard กว้างกว่า NP-Complete และอาจไม่ใช่ปัญหา decision
5. Co-NP เป็นฝั่งส่วนเติมเต็มของ NP

## 4) สรุปคำตอบ
กลุ่มปัญหาทั้งห้านี้ใช้จำแนกความยากเชิงคำนวณของปัญหา โดยจุดสำคัญคือ NP-Complete เป็นแกนกลางของความยากใน NP และ NP-Hard เป็นกลุ่มที่ครอบคลุมปัญหาที่ยากกว่า/กว้างกว่า NP-Complete
